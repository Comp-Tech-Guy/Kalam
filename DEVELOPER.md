# Kalam вҖ” Developer Documentation

## Overview

Kalam is a desktop **profile manager** for Windows customization tools. Users create profiles that bundle configurations for Rainmeter, YASB, wallpapers, GlazeWM, Zebar, and Windhawk. A single click applies everything at once.

**Tech stack:**
- **Desktop shell:** Tauri v2 (Rust backend + WebView frontend)
- **Frontend:** React 19 + Vite + React Router 7
- **System logic:** Python 3 sidecar (compiled to standalone .exe via PyInstaller)
- **Storage:** JSON files in `%APPDATA%/Kalam/`

## Data Schema

### `userSettings.json`
Stores global paths and the currently active profile.

### `userProfiles.json`
Stores the "target state" for each profile.
```json
{
  "Windhawk-Mods": [
    {
      "id": "windows-11-taskbar-styler",
      "enabled": 1,
      "settings": {
        "theme": "futuristic Taskbar",
        "controlStyles[0].target": "..."
      }
    }
  ]
}
```

### `windhawkManifest.json`
Auto-generated by sidecar scan. Catalogs installed Windhawk mods.
```json
{
  "installedMods": [
    { "id": "taskbar-labels", "name": "taskbar-labels", "enabled": 1 }
  ]
}
```

## Windhawk Integration Details

Windhawk is a Windows 11 taskbar customization tool. Kalam manages mods per profile via registry manipulation.

### Registry Layout (Installed/Service mode)

```
HKLM\SOFTWARE\Windhawk\Engine\Mods\
  в”ңв”Җв”Җ {ModID}/
  в”Ӯ   в”ңв”Җв”Җ Disabled          REG_DWORD   (1=disabled, 0=enabled)
  в”Ӯ   в”ңв”Җв”Җ SettingsChangeTime REG_QWORD   (Unix timestamp, triggers reload)
  в”Ӯ   в””в”Җв”Җ Settings/          KEY
  в”Ӯ       в””в”Җв”Җ (mod settings as registry values)
```

Key details from Windhawk developer (discussions #73, #155, #393):
- `Disabled=1` = mod is disabled, `Disabled=0` or absent = enabled
- Settings live in a `Settings` subkey, not as flat values
- `SettingsChangeTime` QWORD triggers Windhawk to reload settings when changed
- The service name is `WindhawkEngine`

### Scan Flow

1. Frontend calls `SideCar("scan")`
2. Python reads `HKLM\SOFTWARE\Windhawk\Engine\Mods` (falls back to `HKLM\SOFTWARE\Windhawk\Mods` then `HKCU` equivalents)
3. For each subkey (mod ID): reads `Disabled` value (inverts вҶ’ `enabled`), reads `Settings` subkey values
4. Writes result to `windhawkManifest.json` in app data folder
5. Frontend reads manifest to populate mod toggle UI
6. No admin needed (read-only registry access)

### Apply Flow

1. Frontend calls `SideCar(profileId)` with target profile
2. Python reads profile's `Windhawk-Mods` array + installed mods from `windhawkManifest.json`
3. Merges states: profile mods get their state, any installed mod NOT in the profile gets disabled
4. If no mods in profile + no mods installed вҶ’ nothing to do, return
5. Generates a `.reg` file (UTF-16 LE with BOM, CRLF) with all changes:
   - `Disabled` DWORD for each mod (inverted from profile `enabled`)
   - `SettingsChangeTime` QWORD set to current timestamp
   - `Settings` subkey with all setting values
6. Writes a `.bat` file (`Kalam_Update_Windhawk_Settings.bat`) that runs:
   - `reg import` the .reg file
   - `net stop WindhawkEngine`
   - `net start WindhawkEngine`
7. Elevates the .bat via `ShellExecuteExW(runas)` вҖ” single UAC prompt
8. Waits for completion, cleans up both temp files

### Profile Switching Behavior

- Always calls `apply_windhawk_profile()` regardless of whether profile has mods
- Profile WITH Windhawk-Mods вҶ’ only those mods enabled, all other installed mods disabled
- Profile WITHOUT Windhawk-Mods вҶ’ all installed mods disabled
- This ensures reverting when switching away from Windhawk profiles

### UAC Elevation Design

- Only triggers for Windhawk HKLM writes (Installed mode only)
- Scan and other features never elevate
- Single UAC prompt via `Kalam_Update_Windhawk_Settings.bat`
- Temp files cleaned up in `finally` block
- Portable mode uses `HKCU` вҶ’ no admin needed

### Portable Mode

- Uses `HKCU\SOFTWARE\Windhawk\Engine\Mods` (no admin)
- Direct `winreg` writes instead of .reg file
- Kills and restarts Windhawk process instead of managing service
- Configured via `userSettings.json`: `Windhawk-Type: "Portable"`, `Windhawk-Path`

### Files Involved

| File | Purpose |
|------|---------|

## Build & Deploy

### Building the Sidecar
```powershell
cd sidecar
python -m PyInstaller kalam-Sidecar-x86_64-pc-windows-msvc.spec
```

This produces `dist/kalam-Sidecar-x86_64-pc-windows-msvc.exe`. Copy it to:
- `app/src-tauri/binaries/my-sidecar/kalam-Sidecar-x86_64-pc-windows-msvc.exe`
- `app/src-tauri/target/debug/kalam-Sidecar.exe` (for dev builds)

### Spec file settings
- `console=False` вҖ” no terminal window
- `upx=True` вҖ” compress with UPX
- `hiddenimports=['winreg']` вҖ” ensure winreg is bundled

## Troubleshooting

- **Scan returns empty mods**: Run Windhawk scan from the app once first
- **Apply doesn't change mods**: Check that UAC prompt was accepted вҖ” without elevation, HKLM writes silently fail
- **Multiple UAC prompts**: Should be exactly one per apply. If more, the restart wasn't chained with the reg import
- **.reg file not found**: TEMP directory might have special chars; the `tempfile.gettempdir()` fallback handles this
